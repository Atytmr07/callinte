# Core Web Design Heuristics

This skill provides 6 actionable, algorithmic heuristics for designing exceptional web interfaces. Apply these principles to ensure designs are intentional, cohesive, and deeply engaging.

## 1. Advanced Typography & The "Anchor" Principle

Typography dictates the core personality of the design.

- **Select the Headline First:** Always choose the headline font before any other typography. The headline acts as the "anchor" dictating the personality (e.g., modern, technical, warm) and matching the core brand.
- **Provide Supporting Contrast:** Ensure all subsequent body or subheading fonts come from the same "world" as the anchor but provide clear visual contrast.
  - *Avoid:* Confusingly similar fonts (e.g., Georgia and Times New Roman).
  - *Example:* Pair a super-condensed headline font (like Instrument Serif) with a spread-out, wider font (like Geist Sands) for subheadings.
- **Use Typography Levels:**
  - *Level 1*: Single font.
  - *Level 2*: Super families (sans-serif, serif, and mono built to gel together).
  - *Level 3*: Combining different font families from different foundries to mesh seamlessly.
- **Find Pairings:** Utilize resources like "Fonts in Use" to find proven pairings for the chosen anchor font.

## 2. The "Star of the Show" (Focal Point)

Interfaces must evoke emotion and be memorable.

- **Create an Emotional Anchor:** Designate a specific "star of the show" (e.g., a specific font, stunning image, or interactive object) that immediately captures attention.
- **Plant the "Seed":** Ensure the focal point acts as a "seed" deeply connected to the product's overall story or function. Do not choose visual elements simply because they look "cool."
- **Example:** For a SaaS product managing complex data, conceptually reference data using an abstract image with vertical lines and gradients evoking dashboards and charts.

## 3. Visual Rhyming (Cohesion & Repetition)

Create a unified aesthetic universe through intentional repetition.

- **Extract Details:** Extract specific visual elements from a strong core focal point (like the brand logo or "star of the show").
- **Apply Consistently:** Apply these extracted details (shapes, colors, gradients, textures) across standard UI components.
- **Examples:**
  - If a brand logo contains curved shapes, apply those same curves to button overlays and directional arrows.
  - If a logo features triangles, replace standard rectangular callout boxes and default plus (+) icons with triangular shapes or masks.

## 4. Depth and Tangibility

Prevent interfaces from feeling flat and strictly digital.

- **Bridge the Digital Gap:** Add tangibility so the interface feels like it exists in the real world.
- **Introduce Subtle Depth:** Use subtle background textures (e.g., noise effects) or 3D glass elements on navigation cards to break up flat pixels.
- **Keep it Background:** Ensure these effects remain strictly subtle. They must act only as real-world background details and never compete for attention with the "star of the show."

## 5. Opacity-Driven Visual Hierarchy

Use opacity, not just size or weight, to establish intuitive visual hierarchy.

- **Avoid 100% Uniformity:** Never use 100% opacity for all text on a page.
- **Standardize Opacity Values:**
  - **100% Opacity:** Main Headlines (highest emphasis, demands immediate attention).
  - **87% Opacity:** High Emphasis Text.
  - **60% to 70% Opacity:** Medium Emphasis / Subheadings.
- **Apply Intentionally:** Use these values to clearly define what the user should read first, what they can skim, and what they can ignore.

## 6. Radical Iteration

Force creative breakthroughs by breaking thought patterns.

- **Do Not Just Tweak:** Refrain from merely adjusting or tweaking the first draft of a design until it looks "better."
- **Test Extremes:** Force the creative process by doing something completely different. Test entirely new layouts, switch between light and dark modes, or choose a completely different element as the "star of the show."
- **Generate Multiples:** Generate multiple (e.g., 12) distinct versions of the focal element to ensure the final choice is the absolute best iteration, rather than settling for the first acceptable draft.
